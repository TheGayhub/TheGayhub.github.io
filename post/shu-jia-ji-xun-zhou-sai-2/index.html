A题(树状数组)

> A - Japan POJ - 3067

题意：给出一些线段，寻找交点的个数。

刚开始看到数据只有1000，想暴力找交点，线段(x,y)(x,y)(x,y),两条线段有交点就是**(x1>x2&&y1<y2||x2>x1||y2<y2)** ,暴力寻找然后超时了。由于是交点只与点的关系所决定，我们不妨按照xxx从大到小开始遍历，我们可以发现，比如说遍历到了当前的 xxx,这个xxx对答案的贡献是多少呢？ 显然就是前面的已经遍历过的那些线段中yyy坐标比当前yyy坐标小的线段的数量，这就可以树状数组来求个数。按照遍历的顺序，依次把yyy插入树状数组中，然后对于当前遍历的累加贡献即可。注意：要用 longlonglong longlonglong 主要代码👇

bool operator<(const P &o)const
{
	if (x == o.x) return y>o.y;
	return x > o.x;
}

B题(找规律)

> B - Paths in a Complete Binary Tree

题意：给定起点，按照给定的字符串在图上走，求终点的节点编号。

这题之前有点印象，和lowbitlowbitlowbit有关。假设当前节点为xxx。

  • xxx是奇数，该节点为叶子节点.
  • xxx为偶数，左儿子就是 x−(lowbit(x)>>1)x-(lowbit(x)>>1)x−(lowbit(x)>>1),右儿子就是 x+(lowbit(x)>>1)x+(lowbit(x)>>1)x+(lowbit(x)>>1).
  • xxx的父亲节点为 x+lowbit(x)x+lowbit(x)x+lowbit(x) 或者 x−lowbit(x)x-lowbit(x)x−lowbit(x).

当时我就卡在如何求父亲节点，因为左右儿子的求法不一样，比赛的时候看图随意口胡了一种判断方法，交上去WA了。第二天补题的时候发现，判断左儿子的方法：直接 v=x+lowbit(x)v=x+lowbit(x)v=x+lowbit(x)，再v=v−(lowbit(v)>>1)v=v-(lowbit(v)>>1)v=v−(lowbit(v)>>1)，判断v是否和x一样即可。比赛时没能想到，确实有点气人啊。
